com.waoss.enesys.cpu

Class CentralProcessor

  • java.lang.Object
    • com.waoss.enesys.cpu.CentralProcessor
  • All Implemented Interfaces:
    java.lang.Cloneable

    public final class CentralProcessor
extends java.lang.Object
implements java.lang.Cloneable

    An instance of this class represents the CPU of the NES The CPU of the NES had only 6 registers.All of them,except the ProgramCounter were 8 bits wide. Each and every instruction of the NES is implemented as a method that takes in an Instruction.The instruction object contains all the required data to execute the instruction.Every method also returns a boolean which is true, if and only if any change to the program counter was done.The process(Instruction) method takes in an instruction object and finds the method to invoke using reflection. All shit(processing) happens in another thread.Which is stored in an atomic reference for thread safety.It is strongly recommended that the run() method not be used unless for debugging, for it runs stuff in the thread that calls it and the loop is infinite unless it be stopped by the interruptThread() method.

      • process

        public void process(@NotNull
                    @NotNull Instruction instruction)

        Processes an instruction. Invokes the required method for the instruction implementation.The method to invoke is found dynamically using reflection. All instruction implementation methods have the same name as the instruction in lower case.The instruction name from the instruction object given as a parameter is converted to lowercase using String.toLowerCase().Then a method of that name is invoked.The method is still incomplete as there is no complete handling of the updation of the program counter.

        Parameters:
        instruction - The instruction
